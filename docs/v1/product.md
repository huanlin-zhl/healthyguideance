# 产品定位与功能

- 主文档：[design.md](design.md)

## 1. 一句话定位

**批量喂截图 → AI 看趋势 → 给出未来几天该怎么吃、怎么练的建议。**

不是记账工具，不是图鉴，是**教练**。截图只是数据入口，**输出永远是建议**。

## 2. 目标用户

- 作者本人，**单用户**
- 不做多账号 / 不做分享 / 不做云同步

## 3. 核心价值与目标

- **健康目标**：在保证健康的前提下，降低体重、体脂率、内脏脂肪
- **使用频率**：体脂秤 1-3 天一次；运动每次结束截图；饮食偶尔写一两句
- **输出形态**：手动触发的趋势分析报告

## 4. 数据类型范围

| 类型 | 来源 | 是否做 |
|---|---|---|
| 运动截图 | 华为运动健康等 app | ✅ |
| 体成分截图 | 体脂秤 app | ✅ |
| 饮食备注 | 用户文字输入（一两句话） | ✅ |
| 食堂菜单截图 | —— | ❌ 不做 |
| 饮食结构化录入 | —— | ❌ 不做 |

## 5. 功能优先级

### 5.1 P0 —— 必须有

1. **批量导入截图**
   - 拖拽 / 文件选择，一次几张到几十张
   - GPT-4o 自动判断截图类型，无需手动选
   - 按图片 SHA256 去重
2. **AI 解析 + 落库**
   - 每张图过一次 GPT-4o，输出符合 schema 的结构化 JSON
   - 失败的图标红提醒，不静默丢
   - 原图与解析结果均存本地
3. **饮食备注（轻量存档）**
   - 用户偶尔写一两句话记录吃了什么
   - 只存档，不抽字段、不去重
   - 录入不调 LLM；分析时整段拼进 prompt
4. **趋势分析报告（核心功能）**
   - 用户点按钮 → 选时间窗口 → 生成报告
   - 手动触发，不做定时
5. **历史记录浏览**
   - 时间轴/列表查看每条数据
   - 浏览历史报告（缓存复用，不重复调模型）

### 5.2 P1 —— 早晚要做

6. 趋势可视化（体重 / 体脂率 / 内脏脂肪折线；运动时长 / 消耗周柱图）
7. 目标设置参与 prompt
8. 报告对比（本期 vs 上期）
9. 数据手动编辑

### 5.3 P2 —— 锦上添花

10. 导出报告（PDF / Markdown）
11. 提醒（该称体重 / 该生成新报告）
12. 鸿蒙端

## 6. 不在范围内

明确**不做**的事：

- 多账号 / 云同步 / 跨设备共享
- 用药 / 医疗建议
- 社交分享 / 排行榜
- 食物图像识别 / 卡路里精确计算
- 自动推送 / 通知（鸿蒙端可能再议）
- 多语言（中文一种）
