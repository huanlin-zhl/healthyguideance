# 错误与异常处理

- 主文档：[design.md](design.md)
- 相关：[ui-flow.md](ui-flow.md)（错误对话框）、[storage.md](storage.md)（failed/ 目录）、[security.md](security.md)（脱敏规则）

## 1. 总原则

个人使用场景，**极简**：

- 出错就弹对话框
- 让用户看懂发生了什么
- 提供"重试" / "改配置" / "取消" 中的合适选项
- 不丢数据（中途失败的状态可恢复）

不做：复杂的退避算法、错误码分级、自动恢复、错误上报。

## 2. 错误分类

粗分 4 类：

| 类别 | 触发场景 | UI 表现 |
|---|---|---|
| **凭证错误** | HTTP 401 / 403、`secrets.dat` 缺失或解密失败 | 弹对话框 + "去设置"按钮 |
| **网络错误** | DNS 失败、连接超时、HTTP 5xx、HTTP 429 | 弹对话框 + "重试" |
| **模型返回错误** | LLM 返回 `kind=unknown` / Schema 校验失败 / 超出预期内容 | **不弹**对话框；写入 `failed/`，记录页标红 |
| **本地 IO 错误** | 磁盘写入失败、文件被占用、路径不存在 | 弹对话框 + "重试"；不写入任何文件 |

## 3. 重试策略

### 3.1 自动重试

仅 HTTP 5xx 和 429，**自动重试 1 次**，间隔 2 秒。再失败即归类为"网络错误"弹对话框。

不做指数退避、不重试多次。

### 3.2 手动重试入口

所有用户主动触发的操作都提供"重试"按钮：

| 操作 | 失败后的重试入口 |
|---|---|
| 导入截图（单张失败） | 导入完成对话框中的"失败列表"，单条"重试"按钮 |
| 导入截图（整批崩溃） | 整个对话框底部"全部重试" |
| 解析失败的历史记录 | 记录页失败行右侧"重试"按钮 |
| 生成报告 | 错误对话框中"重试" |
| 设置保存 | 错误对话框中"重试" |
| 备注保存 | 错误对话框中"重试" |

### 3.3 解析失败的特殊处理

截图被 GPT 返回为 `unknown` 或 schema 校验失败时：

- **不弹**错误对话框（批量导入时频繁弹会很烦）
- 该截图写入 `failed/<yyyy-MM>/<id>/`，附 `error.json`（结构见 [data-model.md §8](data-model.md)）
- 在导入完成对话框的汇总里计入"失败 X 张"
- 用户事后在记录页（筛选"失败"）查看 + 重试

## 4. 错误对话框规范

### 4.1 统一样式

```text
┌──────────────────────────────────────────┐
│ ⚠ 操作失败                          [×]  │
├──────────────────────────────────────────┤
│ <一句话说明>                              │
│                                          │
│ 详细信息：                                │
│ ┌──────────────────────────────────────┐ │
│ │ <错误细节，可滚动>                   │ │
│ └──────────────────────────────────────┘ │
│                                          │
│              [取消]  [次要按钮]  [主按钮] │
└──────────────────────────────────────────┘
```

- 标题图标 ⚠
- 第一行：用户能懂的一句话
- 详细信息：折叠区域，默认收起，点开看技术细节（便于用户截图反馈）
- 按钮：**主按钮始终是"重试"或"去设置"**，次要按钮按场景定，"取消"始终存在

### 4.2 各类错误的文案约定

**凭证错误（401 / 403）**：

```
标题：凭证无效
说明：Azure OpenAI 拒绝了当前 API Key。请检查 Key 是否正确，或前往设置页重新填写。
按钮：[取消] [去设置]   ← 点"去设置"自动打开设置页并聚焦到 API Key 输入框
```

**凭证缺失 / 解密失败**：

```
标题：未配置 API Key
说明：未找到可用的 API Key，请先完成配置。
按钮：[去配置]   ← 自动打开首次配置对话框
```

**网络错误（连接失败 / 5xx / 429 自动重试后仍失败）**：

```
标题：连接 Azure OpenAI 失败
说明：网络不可达或服务暂时不可用，请稍后重试。
详细信息：HTTP 503 / Endpoint: https://xxx...openai.azure.com
按钮：[取消] [重试]
```

**本地 IO 错误**：

```
标题：本地存储错误
说明：写入数据时出错，请检查磁盘空间或文件权限。
详细信息：<异常 Message>
按钮：[取消] [重试]
```

**模型返回不在预期**（用户手动操作触发，而非批量导入）：

```
标题：模型返回异常
说明：AI 未能识别这张截图。可能是图片内容不属于运动或体成分截图，或图片质量过低。
按钮：[取消] [重试]
```

## 5. 数据不丢原则

中途失败时的状态保证：

| 操作 | 失败时的状态 |
|---|---|
| 导入截图（解析失败） | 截图已落入 `failed/`，原图保留；重试有数据可用 |
| 导入截图（本地 IO 失败） | 截图未落盘；用户需重新选择文件 |
| 生成报告（LLM 失败） | 不写入 `reports/`；上次报告仍可看 |
| 设置保存（写盘失败） | 内存中凭证不更新；上次有效凭证仍可用 |
| 备注保存（写盘失败） | 输入框内容保留，不清空；用户可重试 |

## 6. 日志

### 6.1 位置与滚动

- 路径：`%LocalAppData%\HealthyGuidance\logs\app.log`
- 滚动策略：按天，文件名 `app-yyyy-MM-dd.log`
- 单文件上限 1 MB，超出后追加 `.1` `.2` 序号
- 保留最近 7 天，更早自动删除

### 6.2 日志级别

- `Info`：用户主要操作（导入开始/完成、报告生成开始/完成、设置保存）
- `Warn`：可恢复异常（自动重试触发、解析失败但已归档）
- `Error`：弹对话框前必记录一条

### 6.3 脱敏

严格遵循 [security.md §5](security.md)：

- 不输出完整 API Key
- Endpoint 输出 `https://xxx...openai.azure.com`
- 异常堆栈中如包含 Key 或完整 Endpoint，输出前替换

### 6.4 用户访问日志

- 设置页"数据目录位置"打开后，用户可手动浏览到 `logs/` 子目录
- 不在 UI 中直接展示日志
- 不做"上报错误"功能

## 7. 不做的事

- 不做自动错误上报 / 遥测
- 不做错误码体系（不引入 `HG-E1001` 这类编号）
- 不做错误对话框国际化（中文一种）
- 不做 retry-after 头的解析（429 直接 2 秒后重试 1 次）
- 不做断线重连 / 长连接
