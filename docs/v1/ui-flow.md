# UI 流程

- 主文档：[design.md](design.md)

## 1. 设计语言

遵循 WinUI 3 / Fluent Design：

- **左侧 NavigationView** 作为主导航
- **CommandBar 在标题栏右侧** 放全局动作
- **Mica 材质背景** + 圆角控件
- 设置入口为标题栏右上角齿轮图标（不进左导航）

## 2. 应用主框架

```text
┌─────────────────────────────────────────────────────────┐
│ [HealthyGuidance]              [+ 导入] [+ 备注] [⚙]    │
├──────────┬──────────────────────────────────────────────┤
│ 概览     │                                              │
│ 记录     │           内容区（随导航变化）               │
│ 备注     │                                              │
│ 报告     │                                              │
└──────────┴──────────────────────────────────────────────┘
```

- 左侧导航：4 个一级入口（概览 / 记录 / 备注 / 报告），默认选中"概览"
- 标题栏右侧 CommandBar：`+ 导入截图`、`+ 写备注`、`⚙ 设置` 三个按钮，**任何页面都可见**
- 全局动作按钮的点击均触发模态对话框，不跳页

## 3. 概览页（默认页）

进入应用看到的第一屏。布局自上而下：

### 3.1 最近一次分析报告卡片

- 显示：报告生成时间、所用时间窗口、核心结论摘要、`查看 →` 链接
- 点 `查看 →` 跳报告详情
- **无报告时**：显示"还没有分析报告"提示 + `生成第一份` 按钮（点击触发"生成报告"流程，见 §7.2）

### 3.2 最近记录时间线

- 展示**最近 10 条成功记录**（运动 + 体成分混排，按时间倒序）
- **不显示失败记录**
- 单行内容：类型图标、事件时间、核心字段摘要（如 `骑行 / 30.85km / 861kcal`）
- 单条点击 → 跳"记录详情"页
- 底部 `全部 →` 跳"记录"页
- **无记录时**：显示"还没有数据"提示 + `导入第一张截图` 按钮

## 4. 记录页

### 4.1 列表

- 完整时间轴，按时间倒序
- 顶部筛选条：`全部 / 运动 / 体成分 / 失败` 四个标签
- 单条点击 → 详情页

### 4.2 失败记录的视觉

- 整行**标红**
- 行内显示简短错误原因
- 鼠标悬停显示完整错误信息（tooltip）
- 行尾有 `重试` 按钮
- 点 `重试` → 重新调用 GPT-4o 解析；成功后该条从 failed 移到 records，列表自动刷新

### 4.3 记录详情页

- 上方：原始截图（可放大查看）
- 下方：解析后的 JSON 字段（结构化展示，不是裸 JSON）
- 字段允许编辑（P1 功能，UI 留位置但 P0 可只读）
- 右上角操作：`删除`

## 5. 备注页

### 5.1 输入区（常驻顶部）

- 多行文本框 + 时间选择器（默认当前时间，到分钟）
- `保存` 按钮
- 保存后立即追加到列表顶部，无需刷新

### 5.2 历史列表

- 按月分组（折叠/展开）
- 当月默认展开，往月默认折叠
- 单条显示时间戳 + 原文
- 单条右侧操作：`编辑` / `删除`

## 6. 全局动作按钮

### 6.1 `+ 导入截图`

点击 → 弹模态对话框：

```text
┌──────────────────────────────────────────┐
│ 导入截图                            [×]  │
├──────────────────────────────────────────┤
│                                          │
│   ┌──────────────────────────────────┐   │
│   │                                  │   │
│   │      拖拽图片到此处              │   │
│   │      或 [选择文件]                │   │
│   │                                  │   │
│   └──────────────────────────────────┘   │
│                                          │
│   已选 0 张                              │
│                          [取消] [开始]   │
└──────────────────────────────────────────┘
```

点 `开始` 后对话框内嵌进度：

```text
正在解析 3/7 ...
[████████░░░░░░░░░░░░] 43%
```

完成后内嵌结果摘要：

```text
✓ 成功 6 张
✗ 失败 1 张
（点击查看失败列表）
                              [关闭]
```

关闭对话框后，如果当前在"概览"或"记录"页则自动刷新。

### 6.2 `+ 写备注`

点击 → 弹小型对话框：

```text
┌──────────────────────────────────────────┐
│ 写备注                              [×]  │
├──────────────────────────────────────────┤
│ 时间：[2026-05-31 14:30 ▼]                │
│                                          │
│ ┌──────────────────────────────────────┐ │
│ │                                      │ │
│ │  （输入内容）                        │ │
│ │                                      │ │
│ └──────────────────────────────────────┘ │
│                          [取消] [保存]   │
└──────────────────────────────────────────┘
```

保存后关闭对话框，如果当前在"备注"页则自动刷新。

### 6.3 `⚙ 设置`

详见 §8。

## 7. 报告页

### 7.1 列表

- 报告卡片列表，按生成时间倒序
- 单卡显示：生成时间、所用时间窗口、结论一句话摘要
- 点击 → 报告详情

### 7.2 生成新报告流程

报告页顶部按钮 `+ 生成新报告` → 弹时间窗口选择对话框：

```text
┌──────────────────────────────────────────┐
│ 生成报告                            [×]  │
├──────────────────────────────────────────┤
│ 选择时间窗口：                            │
│                                          │
│  [ 过去 7 天  ] [ 过去 30 天 ] [ 过去 90 天 ] │
│  [ 自定义日期范围 ]                       │
│                                          │
│                          [取消] [生成]   │
└──────────────────────────────────────────┘
```

- 4 个预设按钮 + 自定义范围
- **记住上次选择**，下次打开默认高亮上次选项
- 点 `生成` → **模态进度对话框**（同步等待，转圈 + 文字"正在生成报告..."）
- 成功 → 关闭进度对话框 → 自动打开新报告详情页
- 失败 → 关闭进度对话框 → 弹错误对话框（错误信息 + `重试` / `取消` 按钮）

## 8. 设置页

包含项：

| 项 | 类型 | 说明 |
|---|---|---|
| API Key | 文本框（密码框） | 修改 Azure OpenAI key；保存时 DPAPI 加密写盘 |
| 默认时间窗口 | 下拉选择 | 7 / 30 / 90 天；首次启动默认 30 天 |
| 数据目录位置 | 只读文本 + `打开` 按钮 | 展示 `%LocalAppData%\HealthyGuidance\` 完整路径，点 `打开` 用资源管理器打开 |
| 目标体重 | 数字输入 | 单位 kg；分析时随 prompt 一起传给 GPT |
| 目标体脂率 | 数字输入 | 单位 %；分析时随 prompt 一起传给 GPT |
| 关于 / 版本 | 文本展示 | 应用版本号、构建时间 |

## 9. 首次启动

应用启动时检查 `config/secrets.dat` 是否存在：

- **不存在** → 跳"首次配置"对话框，强制用户填 API Key 才能进主界面
- **存在** → 直接进概览页

首次配置对话框：

│ └──────────────────────────────────────┘ │

## 10. 页面跳转关系

```text
                  ┌─────────────┐
启动 ───┬───► 首次配置（首次）   │
        │         └─────────────┘
        ▼
   ┌─────────┐  ◄───── + 导入截图（对话框，任意页可触发）
   │ 概览页  │  ◄───── + 写备注（对话框，任意页可触发）
   └────┬────┘  ◄───── ⚙ 设置（对话框/独立页，任意页可触发）
        │
        ├──► 报告卡片"查看"───► 报告详情页
        ├──► 最近记录单条 ────► 记录详情页
        ├──► "全部" ────────► 记录页
        │
   左导航：
        ├──► 概览页
        ├──► 记录页 ─────────► 记录详情页
        ├──► 备注页
        └──► 报告页 ─────────► 报告详情页
                  └───► + 生成新报告 ─► 报告详情页
```
